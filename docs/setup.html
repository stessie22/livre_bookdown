Chapter 2 Connecter Bookdown à Github

2.1 Démarche actualisée - 24-03-2022

  1. Go to your GitHub account: github.com/username
  2. Click on Repositories, and the green button “New” to create a new repo
  3. Name this new repo “mon_livre”
  4. DO NOT initialize this repo with a README
  5. Click the green “create repository” button — this will take you to your new repo
  6. Click the tiny “clone or download” button near the top and COPY URL

https://github.com/oswaldoforey/mon_livre.git

  1. In the Console, type usethis::use_git() and say Yes to the two prompts. This will restart R and give you a new Git tab in the upper right pane.
  2. Now, click on the Terminal tab next to the Console tab.
  3. Type git remote add mon_livre https://github.com/oswaldoforey/mon_livre.git
  4. Type git push –set-upstream mon_livre master
  5. Go back to github.com/username/“mon_livre” and refresh — our files should be there! But we want it to be a book published as https://username.github.io/mon_livre.
  6. Click ‘Settings’ (la roue dentée en haut à droite sous Pin)
  7. sur le coté gauche cliquer dans ‘Pages’
  8. Change the Source pulldown from “None” to “master branch /docs folder”
  9. It should say “Your site is ready to be published at https://username.github.io/mon_livre/” — click the link to see but error page 16.effacer le readme, le bouton environnement en bas à droite de la page avec une fusée apparait
  10. cliquer dessus ça marche (il faut attendre un peu des fois et recharger la page)

2.2 Moving forward

As you write your .Rmd files, build the book and commit all files, including the docs/ folder, and your published book will be updated!
